# Bordshockey.net Scraper

This project scrapes table hockey tournament results from [bordshockey.net](https://bordshockey.net) and normalizes the data into a structured CSV format. It also includes a player mapping system to link scraped names to a canonical player database.

## Features

- **Automated Scraping:** Fetches tournament data including match results, stages, and dates.
- **Incremental Updates:** Checks existing data to only scrape new tournaments.
- **Player Mapping:** Maps scraped player names to canonical PlayerIDs using exact matching, fuzzy matching (difflib), and manual overrides.
- **GitHub Actions:** Automated weekly workflow to keep the dataset up-to-date.

## Files

- `bordshockey_scraper.py`: The main scraping script.
- `map_players.py`: Script to generate `player_id_map.csv` by matching `players.csv` against `players_data.csv`.
- `manual_mapping.json`: JSON file for manually correcting or ignoring player mappings.
- `bordshockey_results.csv`: The primary output dataset containing all match results.
- `player_id_map.csv`: The generated mapping of ScrapedName -> PlayerID.
- `players.csv`: List of unique player names extracted from the results (generated by scraper).
- `unmapped_players.csv`: List of players that could not be confidently mapped.

## Usage

### Prerequisites

```bash
pip install -r requirements.txt
```

### Running the Scraper

To scrape new tournaments and append them to the results:

```bash
python bordshockey_scraper.py
```

### Updating Player Mappings

To regenerate the player mapping file (e.g., after updating `manual_mapping.json`):

```bash
python map_players.py
```

## Configuration

### Manual Mapping

Edit `manual_mapping.json` to handle specific cases:
- **Map to ID:** `"Scraped Name": 12345`
- **Ignore/No Match:** `"Scraped Name": null` (Prevents fuzzy matcher from guessing incorrectly)

## Automation

The `.github/workflows/scrape_weekly.yaml` workflow allows this scraper to run automatically every Monday at 18:00 UTC. It commits any new data and mappings back to the repository.
